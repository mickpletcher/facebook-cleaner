# Project Instructions

- Update `CHANGELOG.md` in the same change set every time any project file is added, changed, renamed, or removed.
- Record code, database, matching, reporting, configuration, privacy, test, documentation, and behavior changes.
- Do not consider a change complete until its changelog entry is present.
- Never include private Facebook content, identifiers, media, export paths, database contents, or report contents in the changelog.
- Keep `README.md` accurate and complete whenever behavior, supported sources, commands, requirements, outputs, privacy controls, or operator workflows change.
- Do not consider a user-facing change complete until both `README.md` and `CHANGELOG.md` are synchronized with the implementation.
